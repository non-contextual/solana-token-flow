# Contributing to Solana Token Flow

Thank you for your interest in contributing. This project operates under a strict, minimalist philosophy. 

## Workflow

1. Fork the repository and create your branch from `main`.
2. Ensure your code aligns with the existing architecture. No bloat.
3. If you've added code that should be tested, add tests.
4. Update the documentation if necessary.
5. Submit a pull request.

## Pull Request Process

- Ensure your PR description clearly states the problem and the solution.
- Keep commits atomic and descriptive (e.g., `feat: add Helius RPC retry logic`).
- The PR will be reviewed and either merged or rejected based on utility and alignment with the core system.
